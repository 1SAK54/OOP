# Task_2_3_1

## Описание

Проект реализует многопользовательскую игру "Змейка" на JavaFX с клиент-серверной архитектурой

Клиент отвечает за отображение игры, ввод данных подключения, обработку клавиш и отрисовку поля

Сервер отвечает за хранение состояния игры, движение змей, обработку столкновений, появление еды,
респавн игроков и лидерборд

## Правила игры

Игрок управляет змейкой и собирает еду на поле

За каждую съеденную еду игрок получает очко, а змейка увеличивается в длине

Если змейка сталкивается со стеной, собой или другой живой змейкой, 
она умирает и возрождается через 3 секунды

После смерти счёт игрока сохраняется

Если игрок отключается от сервера, его змейка удаляется из игры и
больше не отображается в лидерборде

## Управление

- `W` / `↑` - вверх
- `S` / `↓` - вниз
- `A` / `←` - влево
- `D` / `→` - вправо

## Архитектура

Проект разделён на две основные части:

- `client` - JavaFX-клиент
- `server` - серверная часть игры

Основные классы клиента:

- `ClientApp` - запуск JavaFX-приложения
- `ClientLauncher` - launcher для корректного запуска клиента
- `ConnectController` - экран подключения
- `GameController` - управление игровым экраном
- `GameView` - отрисовка игры
- `NetworkClient` - обмен сообщениями с сервером
- `GameState` - состояние игры, полученное от сервера

Основные классы сервера:

- `ServerMain` - запуск сервера
- `SnakeServer` - приём подключений
- `ClientHandler` - обработчик одного клиента
- `MultiplayerGame` - основная логика игры
- `PlayerSnake` - змейка игрока

## Запуск сервера

```bash
./gradlew runServer
```

По умолчанию сервер запускается на порту `5555`

## Запуск клиента

```bash
./gradlew runClient
```

После запуска нужно ввести:

- nickname
- IP сервера
- порт сервера

Для локального запуска:

```text
Host: localhost
Port: 5555
```

## Итог

В проекте реализована сетевая версия змейки, где сервер управляет всей игровой логикой,
а клиент отображает состояние игры и отправляет команды игрока
